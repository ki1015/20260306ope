# オペレーション戦略 理論学習アプリ 計画書

## 1. 資料から抽出した「理論・フレームワーク・考え方」一覧

ケース（ドミノ・ピザ、セブン-イレブン）に依存しない、**汎用の概念**のみを整理しました。

| カテゴリ | 理論・フレームワーク・考え方 |
|----------|------------------------------|
| **ポジショニング** | Value Line分析、ポジショニングマップとの違い（軸がKSFと結びつくか）、トレードオフの2軸での位置づけ |
| **内製・外製** | 内製/外製の2軸（OP上のValueが高い方 × 戦略上重要度）、コア業務で外部活用時の手法（FC、パートナーシップ、シェアドサービス） |
| **パートナーシップ** | Win-Win、相互の「価値」の把握、レベニューシェア＜プロフィットシェア＜戦略的アライアンス、信頼・コスト透明性 |
| **リーン** | リーン生産方式、リーン企業体（Value Chain全体での無駄削減）、同一概念による「価値」の共有 |
| **外部連携** | Value Chain上の連携 vs 同一プロセス上の連携、垂直統合・水平統合 |
| **継続的改善** | インフラ・仕組みの改善 ＋ 人のスキル・意識の改善の両輪、PDCA、設計時に「人の成長」を組み込む |

| **フランチャイズ** | 本部の継続的価値提供、役割分担の戦略的設計、フィー設計（取り逃しのない請求）、フランチャイジーサイズ（小さすぎ/大きすぎ） |

## 2. アプリの目的とコンセプト

- **目的**: ケースの固有名詞や数字を問うのではなく、「どの局面でどのフレームワークを使うか」「トレードオフをどう考えるか」といった**考え方**を定着させる。
- **対象**: オペレーション戦略の授業受講者、MBA/社会人でオペ戦略を学び直したい人。
- **使い方**: 通勤・隙間時間に、資料（PDF）を読みながら or 読んだ後に、理論だけを復習する。

---

## 3. アプリ案：3パターン

### 案A: 「理論クイズ」アプリ（シンプル）

- **機能**: 資料から抽出した理論・フレームワークについて、4択または○×で出題。
- **出題ルール**: 
  - 固有名詞（ドミノ、セブン、数値）は使わない。
  - 「Value Line分析で軸に据えるべきなのは？」「内製すべきなのは2軸のどこ？」「PSIモデルでPは何から導く？」など、定義・使い方・トレードオフの理解を問う。
- **メリット**: 実装が比較的容易。単元（ポジショニング、内製外製、SCM等）でフィルタ可能にすると、苦手分野の復習に使える。

### 案B: 「フレームワーク選択」アプリ

- **機能**: 短いシナリオ（業種・状況の説明のみ、ケース名は出さない）を提示し、「どのフレームワークで整理するか」「次の一手として何を考えるか」を選ばせる。
- **例**: 「小売とメーカーが在庫ロスと機会ロスを減らしたい。関係をどう整理する？」→ パートナーシップ、リーン企業体、相互の価値の把握 など。
- **メリット**: 理論の「いつ使うか」が身につく。資料の演習（パートナーシップ構築など）に近い思考を再現できる。

### 案C: 「用語⇔考え方」カード＋クイズ

- **機能**: 
  - **カード**: 用語（Value Line、PSI、QCD+I、リーン企業体…）をタップすると、「定義」「何と何のトレードオフか」「判断の軸」が短文で表示される。
  - **クイズ**: カードの内容から、穴埋め・並び替え・マッチング（用語と説明）を出題。
- **メリット**: 暗記ではなく「意味の理解」に寄せられる。資料の図（2軸マトリクス等）を簡略化した図で出題すると、視覚的にも定着しやすい。

---

## 4. 推奨：ハイブリッド型

- **メイン**: 案Aの理論クイズ（4択/○×）で、単元別・難易度別に出題。
- **サブ**: 案Cの用語カード（一覧・タップで詳細）を「辞書」として用意し、解けなかった問題からその用語のカードに飛べるようにする。
- **発展**: 余力があれば、案Bのシナリオ問題を「実践」モードとして数問用意する。

これにより、「覚える（カード）→ 試す（クイズ）→ 使う（シナリオ）」の流れを携帯1本で完結できます。

---

## 5. 問題の作り方（理論に絞るルール）

資料から問題を作る際のルール案です。

1. **出さないもの**
   - 企業名・国名・具体的数値（○億ドル、○%、○年）
   - 「ドミノは〇〇した」のようなケース固有の事実

2. **出すもの**
   - フレームワークの定義（「Value Line分析とは？」「PSIのP,S,Iは何の略か」）
   - 2軸の意味（「内製/外製の縦軸・横軸は何か」「どれが内製すべきか」）
   - トレードオフ（「安全在庫とサイクル在庫の違い」「QCD+Iの主なトレード例」）
   - 考え方（「リーン企業体で重要なこと」「パートナーシップ構築の条件」「グローバルとローカルの判断軸」）

3. **問題形式の例**
   - 4択: 「SCMを『構築』と『運用』に分けたとき、在庫の最適化は主にどちらで行うか」
   - ○×: 「Value Line分析の軸には、業界KSFが密接に関連している」
   - 並び替え: 「レベニューシェア、プロフィットシェア、戦略的アライアンスを、連携の深さの順に並べよ」
   - マッチング: 用語（PSI, S&OP, 安全在庫, サイクル在庫）と説明を結びつける

## 6. 技術・実装の方向性

- **プラットフォーム**: 
  - **PWA（Progressive Web App）**: 携帯ブラウザで開き、ホームに追加でアプリのように使える。ネイティブアプリの審査が不要。
  - または **React Native / Flutter**: ストア配布したい場合。
- **データ**: 
  - 問題・選択肢・正解・解説・単元タグをJSONまたはMarkdownで管理。
  - 資料の「理論部分」だけをテキスト化した「出題元」を1問ごとに紐づけておくと、復習時に資料に戻りやすい。
- **機能案**:
  - 単元フィルタ（ポジショニング、内製外製、SCM、在庫、グローバル…）
  - 正答率の記録（ローカル or 簡易バックエンド）
  - 間違えた問題の再出題
  - ダークモード（通勤時の目に優しく）

## 7. 次のステップ

1. **問題リストの作成**: 上記ルールに沿って、資料の理論部分から30〜50問程度の「問題・正解・解説・単元」をリスト化する。
2. **データ形式の決定**: JSONスキーマを決め、1問1ファイル or 1JSONに全問格納するか決める。
3. **プロトタイプ**: まずはPWAで、単元2〜3個・各5問程度でクイズ画面と結果画面だけ作る。
4. **用語カード**: 主要20〜30用語について「用語名・定義・関連トレードオフ・一言」のカードデータを作成する。

この計画をベースに、まずは「問題リストの作成」から進めると、アプリの中身が明確になり、開発しやすくなります。必要なら、実際の問題例を資料のページ番号付きでさらに追加することもできます。
