# アプリをみんなに共有する方法

このアプリは HTML・CSS・JavaScript だけの静的サイトなので、**無料のホスティング**に置くだけで、URL を教えた人全員がブラウザで開けます。パスワードはそのまま「takoyaki」で有効です。

---

## 方法1: Netlify（おすすめ・かんたん）

1. **https://www.netlify.com/** にアクセスし、アカウント作成（無料）。
2. 画面の「**Drag and drop your site output folder here**」のエリアに、**このプロジェクトのフォルダ全体**（`20260305オペ戦` フォルダごと）をドラッグ＆ドロップ。
   - 中身に `app` フォルダと `data` フォルダが入っている状態でドロップする。
3. しばらくすると、`https://○○○.netlify.app` のような URL が表示される。
4. **共有するURL:**  
   `https://○○○.netlify.app/app/`  
   （末尾の **`/app/`** を忘れずに。）
5. 相手には「このURLを開いて、パスワード `takoyaki` を入力してね」と伝える。

※ スマホのブラウザで開いても同じように使えます。

---

## 方法2: GitHub Pages（詳しい手順）

**→ プロジェクトフォルダ内の「GitHubで公開する手順.md」に、リポジトリ作成から Pages 設定まで書いてあります。**

おおよその流れ:
1. **GitHub**（https://github.com）でアカウント作成（無料）。
2. 新しいリポジトリを作成。
3. このフォルダで `git init` → `git add .` → `git commit` → `git remote add origin` → `git push` でコードを送る。
4. リポジトリの **Settings** → **Pages** で「Deploy from a branch」を選び、ブランチ `main` で保存。
5. 数分後に `https://【ユーザー名】.github.io/【リポジトリ名】/app/` で公開される。
6. 共有するときは上記URLとパスワード `takoyaki` を伝える。

---

## 方法3: Vercel

1. **https://vercel.com** でアカウント作成（無料）。
2. 「Add New」→「Project」で、このプロジェクトのフォルダをアップロード（または GitHub と連携してリポジトリを選択）。
3. **Root Directory** はそのまま（プロジェクトのルート）。
4. デプロイ後、表示されたURLのうしろに **`/app/`** を付けたものがアプリのURL。
5. そのURLとパスワードを共有する。

---

## 注意すること

- **アップロードするのは「プロジェクトのルート」全体**です。  
  `app` フォルダと `data` フォルダの両方が、同じ階層にある状態でアップロードしてください。
- 共有するURLは必ず **`/app/`** で終わるようにしてください（例: `https://xxx.netlify.app/app/`）。  
  `index.html` が `app` フォルダの中にあるためです。
- パスワードはクライアント側（ブラウザ）でチェックしているだけなので、**「知っている人だけ使えるようにする」**程度の用途向けです。URLとパスワードは信頼できる人にだけ伝えてください。
